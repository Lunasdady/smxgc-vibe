## 三、⚠️ 生产部署关键注意事项

### 3.1 数据库切换（最重要）

当前开发环境使用的是 **SQLite 文件数据库**（`file:./dev.db`），生产环境**强烈建议**切换到 PostgreSQL 或 MySQL：

```bash
# 生产环境 .env 配置
DATABASE_URL="postgresql://username:password@localhost:5432/smxgc_db?schema=public"
```

**原因**：
- SQLite 是单文件数据库，高并发写入时容易锁表
- 生产环境数据量增长后性能下降明显
- 无法做多实例部署（文件锁冲突）

**迁移步骤**：
```bash
# 1. 修改 schema.prisma 的数据源配置（如需要）
# 2. 更新 .env 中的 DATABASE_URL
# 3. 推送数据库结构
npm run db:push
npm run db:generate
```

---

### 3.2 环境变量配置

必须配置的变量（`.env` 文件或服务器环境变量）：

```bash
# 数据库连接（生产使用 PostgreSQL/MySQL）
DATABASE_URL="postgresql://username:password@host:5432/dbname?schema=public"

# 管理后台密码（务必修改默认值）
ADMIN_PASSWORD="your-strong-password-here"

# JWT 签名密钥（务必修改为强随机字符串）
JWT_SECRET="your-random-secret-key-at-least-32-characters-long"
```

**⚠️ 安全提醒**：
- `ADMIN_PASSWORD` 和 `JWT_SECRET` **绝不可使用默认值**
- `JWT_SECRET` 泄露后攻击者可伪造登录凭证
- 建议 `JWT_SECRET` 使用 `openssl rand -base64 32` 生成

---

### 3.3 导入进度追踪的部署限制（重要）

当前导入进度功能使用 **内存 Map** 存储进度状态：

```typescript
const importProgressMap = new Map();
(globalThis as any).__importProgressMap = importProgressMap;
```

**部署限制**：

| 部署方式 | 支持情况 | 说明 |
|----------|----------|------|
| 单实例部署 | ✅ 完全支持 | 推荐方式 |
| 多实例/负载均衡 | ❌ 不支持 | 轮询请求可能落到不同实例 |
| Serverless (Vercel) | ⚠️ 有限支持 | 实例冷启动会丢失进度 |

**建议**：
- 单服务器部署时无需修改
- 多实例部署时，建议将进度存储改为 **Redis** 或共享数据库表

---

### 3.4 文件上传大小限制

Excel 文件可能较大（全量数据），需要确认以下限制：

```javascript
// next.config.js 中添加
const nextConfig = {
  experimental: {
    serverActions: {
      bodySizeLimit: '10mb',  // 根据实际文件大小调整
    },
  },
};
```

Nginx/反向代理也需要同步调整：
```nginx
client_max_body_size 20M;
```

---

### 3.5 静态资源与构建优化

```javascript
// next.config.js 建议配置
const nextConfig = {
  images: {
    domains: [],
    unoptimized: true,  // 如果使用自有服务器部署，建议关闭图片优化
  },
  output: 'standalone',  // 生成独立部署包（推荐）
};
```

使用 `output: 'standalone'` 后，构建产物会包含最小化的 Node.js 运行时，方便 Docker 部署。

## 四、部署步骤

### 方式一：服务器直接部署（推荐）

```bash
# 1. 拉取代码
git clone <your-repo-url>
cd smxgc-vibe

# 2. 安装依赖
npm ci --production

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入生产环境配置

# 4. 初始化数据库
npm run db:push
npm run db:generate

# 5. 构建
npm run build

# 6. 启动（使用 PM2 等进程管理器）
npm start
# 或使用 PM2：pm2 start npm --name "smxgc" -- start
```

---

### 方式二：Docker 部署

**Dockerfile**：

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --production

COPY . .
RUN npx prisma generate
RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]
```

**docker-compose.yml**：

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/smxgc
      - ADMIN_PASSWORD=your-password
      - JWT_SECRET=your-secret
    depends_on:
      - db

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: smxgc
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

---

### 方式三：Vercel 部署（Serverless）

1. 将代码推送到 GitHub 仓库
2. 在 Vercel 中导入项目
3. 设置环境变量：
   - `DATABASE_URL`（使用 PostgreSQL）
   - `ADMIN_PASSWORD`
   - `JWT_SECRET`
4. 点击部署

**注意**：需要连接外部 PostgreSQL 数据库（如 Vercel Postgres、Supabase），导入进度条功能在 Serverless 环境下可能不稳定。

## 六、常见问题

**Q: 可以只用 SQLite 部署吗？**
> 可以，但仅适合低并发、单用户场景。数据量超过 10 万条后建议迁移。

**Q: 需要 Redis 吗？**
> 当前系统不依赖 Redis。如果未来多实例部署，需要将导入进度 Map 改为 Redis 存储。

**Q: 有消息队列/定时任务吗？**
> 没有。系统是纯 Web 应用，无后台定时任务。如需定时导入数据，建议配合 `cron` + `curl` 调用 API。

**Q: 需要专门的文件存储服务吗？**
> 不需要。Excel 文件在导入时临时读取，不持久化存储。
