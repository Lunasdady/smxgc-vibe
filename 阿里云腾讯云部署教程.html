阿里云 / 腾讯云 服务器部署教程（个人版）

0
准备工作
你需要准备：
  • 一台云服务器（阿里云/腾讯云，2核2G内存即可）
  • 一个域名（可选，没有就用IP访问）
  • 本地电脑已安装 Git（用于上传代码）
  • 项目代码已准备好（本教程假设代码在 G:\vibe coding\smxgc-vibe）

🖥️ 推荐服务器配置

2核 CPU / 2G 内存 / 50G SSD / 3M 带宽
系统：Ubuntu 22.04 LTS
费用：约 100元/年（新用户更便宜）

🌐 域名（可选）

没有域名直接用 IP 地址访问
有域名可以配置 HTTPS 更安全
国内域名需备案（约 7-20 天）

1
购买云服务器

阿里云和腾讯云都有轻量应用服务器，价格便宜、配置简单，适合个人使用。

阿里云购买步骤：

  1. 打开 aliyun.com，登录账号
  2. 搜索"轻量应用服务器"，进入购买页面
  3. 选择配置：
    • 地域：选择离你最近的（如华北2北京、华东1杭州）
    • 镜像：选择 Ubuntu 22.04
    • 套餐：2核2G，50G SSD，3M带宽（约99元/年）
  4. 点击购买，完成支付
  5. 进入控制台 → 轻量应用服务器 → 找到你的服务器
📷 此处为阿里云控制台截图位置
找到你的服务器实例，记录公网 IP 地址
重要：购买完成后，在控制台找到服务器的公网 IP 地址（如 123.45.67.89），后面会频繁用到。同时设置好登录密码或准备好密钥文件。
2
连接服务器 & 安装环境

用 SSH 工具连接到你的云服务器，安装 Node.js 和 PM2。

2.1 连接服务器

Windows 用户推荐使用 Termius 或 Xshell，Mac 用户直接用终端。

# Windows 打开 PowerShell，Mac 打开终端，执行：
ssh root@你的服务器IP

# 示例：
ssh root@123.45.67.89

# 然后输入密码（输入时不会显示任何字符，正常）
Welcome to Ubuntu 22.04.3 LTS

2.2 安装 Node.js

# 更新软件包列表
apt update && apt upgrade -y

# 安装 Node.js 20（LTS 长期支持版）
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 验证安装
node -v
v20.x.x

npm -v
10.x.x

2.3 安装 PM2（进程管理器）

npm install -g pm2

# 验证
pm2 -v
5.x.x

2.4 安装 Nginx（Web 服务器）

apt install -y nginx

# 验证 Nginx 是否运行
systemctl status nginx

# 在浏览器访问你的服务器 IP，应该能看到 Nginx 欢迎页面
http://你的服务器IP
如果浏览器能打开 IP 看到 Nginx 欢迎页，说明服务器环境已经准备就绪！
3
上传项目代码到服务器

把本地项目代码上传到服务器的 /var/www 目录下。

3.1 在服务器上创建项目目录

mkdir -p /var/www/smxgc
cd /var/www/smxgc

3.2 上传代码（两种方式选一种）

方式 A：使用 Git（推荐，如果你有 Git 仓库）

# 在服务器上克隆代码
git clone https://github.com/你的用户名/你的仓库.git .

# 如果没有 Git 仓库，先在本地初始化并推送
# 本地 PowerShell 执行：
cd "G:\vibe coding\smxgc-vibe"
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main

方式 B：使用 scp 命令直接上传（不用 Git）

# 在本地 Windows PowerShell 中执行（注意路径中的空格）：
scp -r "G:\vibe coding\smxgc-vibe\*" root@你的服务器IP:/var/www/smxgc/

# 上传可能较慢，稍等片刻...
注意：如果上传后服务器上没有 node_modules 文件夹，那是正常的（我们本地上传时通常不带上它）。后面会在服务器上执行 npm install。
4
在服务器上安装依赖 & 构建

连接服务器，进入项目目录，安装依赖并构建生产版本。

# 进入项目目录
cd /var/www/smxgc

# 安装项目依赖（会在服务器上创建 node_modules）
npm install

# 安装 Prisma CLI（用于数据库操作）
npm install -g prisma

# 生成 Prisma 客户端
npx prisma generate

# 推送数据库结构（SQLite 文件会自动创建）
npx prisma db push

# 构建生产版本
npm run build

# 构建成功后，你会看到 .next 文件夹
ls -la .next
关于数据库：本教程使用 SQLite（文件数据库），数据保存在服务器上的 dev.db 文件中。个人使用完全足够，无需额外安装 MySQL/PostgreSQL。数据会自动备份在文件中。
5
配置环境变量

修改生产环境的环境变量，特别是密码和密钥必须改掉默认值。

# 编辑 .env 文件
nano /var/www/smxgc/.env

# 把内容改成下面这样（按实际情况修改）：
DATABASE_URL="file:./dev.db"
ADMIN_PASSWORD="这里设置一个复杂密码，至少8位"
JWT_SECRET="这里设置一个随机字符串，至少32个字符"
安全警告：
• ADMIN_PASSWORD 不要用默认的 admin123456
• JWT_SECRET 推荐用命令生成随机字符串：openssl rand -base64 32
• 不要把 .env 文件提交到 Git 仓库 
# 保存文件：按 Ctrl+O，然后回车，再按 Ctrl+X 退出
6
启动应用（PM2 守护进程）

用 PM2 启动 Next.js 应用，这样关闭 SSH 后网站也能持续运行。

# 进入项目目录
cd /var/www/smxgc

# 用 PM2 启动应用（指定端口 3000）
pm2 start npm --name "smxgc" -- start -- --port 3000

# 查看运行状态
pm2 status

# 你会看到 smxgc 状态是 online，说明启动成功
┌─────┬────────┬────────┬─────────┬──────────┐
│ id  │ name   │ status │ cpu     │ memory   │
├─────┼────────┼────────┼─────────┼──────────┤
│ 0   │ smxgc  │ online │ 0.2%    │ 85.4MB   │
└─────┴────────┴────────┴─────────┴──────────┘

# 保存 PM2 配置（服务器重启后自动启动）
pm2 save
pm2 startup
# 执行上面命令后，会输出一条命令，复制执行它即可
此时，你可以在浏览器访问 http://你的服务器IP:3000 看看是否能打开网站！
7
配置 Nginx 反向代理（用 80 端口访问）

现在需要用 80 端口访问，而不是 3000。配置 Nginx 把 80 端口的请求转发到 3000 端口。

# 创建 Nginx 配置文件
nano /etc/nginx/sites-available/smxgc

# 粘贴以下内容：
server {
    listen 80;
    server_name _;  # 如果有域名，改成你的域名

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 允许上传大文件（Excel 导入用）
    client_max_body_size 50M;
}
# 启用配置
ln -s /etc/nginx/sites-available/smxgc /etc/nginx/sites-enabled/

# 删除默认配置（避免冲突）
rm /etc/nginx/sites-enabled/default

# 测试配置是否正确
nginx -t

# 重启 Nginx
systemctl restart nginx
🎉 完成！现在用浏览器直接访问 http://你的服务器IP（不需要加 :3000）即可打开网站！
8
配置 HTTPS（可选但强烈建议）

如果你有域名，建议配置 HTTPS，数据更安全，浏览器也不会显示"不安全"。

8.1 先解析域名到服务器

  1. 登录你的域名服务商（如阿里云域名、腾讯云 DNSPod）
  2. 添加一条 A 记录：主机记录填 @ 或 www，记录值填你的服务器 IP
  3. 等待 5-10 分钟让 DNS 生效
📷 域名 DNS 解析设置截图位置

8.2 安装 SSL 证书（Let's Encrypt 免费证书）

# 安装 certbot
apt install -y certbot python3-certbot-nginx

# 修改 Nginx 配置中的 server_name
nano /etc/nginx/sites-available/smxgc

# 把 server_name _; 改成你的域名，例如：
# server_name smxgc.yourdomain.com;

# 申请证书（把域名换成你的）
certbot --nginx -d smxgc.yourdomain.com

# 按提示操作，选择 redirect（自动把 HTTP 转到 HTTPS）

# 测试自动续期
certbot renew --dry-run
现在访问 https://你的域名 就能看到绿色小锁标志了！证书会自动续期，无需手动维护。

常见问题排查

问题 1：访问 IP 显示不了网站

# 检查防火墙是否放行了 80 和 3000 端口
ufw status
ufw allow 80
ufw allow 443
ufw allow 3000

# 阿里云/腾讯云控制台也有安全组，需要去网页控制台放行端口

问题 2：npm install 报错

# 可能是内存不足，增加 swap 空间
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

问题 3：上传 Excel 失败

# 检查 Nginx 的 client_max_body_size 是否设置足够大
# 检查 .env 中的 DATABASE_URL 是否正确
# 检查 pm2 logs 看具体错误

问题 4：页面加载很慢

# 可能是带宽不足，Next.js 首次加载需要下载 JS
# 建议开启 Nginx gzip 压缩
nano /etc/nginx/nginx.conf
# 找到 gzip 相关配置，确保 gzip on;
✓
